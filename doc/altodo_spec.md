# altodo フォーマット仕様書

## ライセンス

__altodo__ は __Markdown__ および __CommonMark__ フォーマット仕様の独自拡張である。 Markdown フォーマットについて https://daringfireball.net/projects/markdown/ 参照すること。 CommonMark フォーマットについては https://commonmark.org/ を参照すること。
altodo フォーマット仕様（独自拡張部分）には MIT ライセンスが適用される。詳細は `doc/LICENSE-MIT` を参照。


### 参考と謝意

## 1. 概要

`altodo` は Markdown (CommonMark) ファイル上に埋め込み可能な TODO フォーマットである。
Markdown 内に記載ができるフォーマットの前提であるが、Markdown をベースとした独自拡張であるため、このフォーマットのタスクを記入したテキストファイルを "altodo" ファイル（フォーマット）と呼ぶ。
altodo のファイルは、拡張子 `altodo` のテキストファイルである。

フォーマットの特徴は以下の通り。

- Markdown (CommonMark) の独自拡張としてのテキストフォーマット仕様
    - altodo のフォーマット以外は Markdown (CommonMark) のフォーマット仕様に従う
- TODO は無限にネスト可能
- 1 行および複数行コメントを表現可
- タグ、スター、優先度、開始日・期日、依存関係タグを付与可


## 2. サンプル

以下、Markdown (CommonMark) ファイル内に altodo フォーマットのテキストを記載したサンプルである。

```
# altodo は Markdown 内に記載のできる TODO フォーマットである

前後がタスク行でなければ、 __通常の Markdown (CommonMark) 同様のテキスト__ を記載できる。

[ ] ここから TODO 行
[ ] __太字__ ~~斜線~~ のように、インラインの `Markdown` 記法をタスク内に埋め込める
[x] ここまで TODO 行

TODO 行、およびコメント行以外は通常の Markdown (CommonMark) のフォーマットである。


# 見出しにも #tag を埋め込むことができる

## altodo ファイル例 #tag

以下、altodo フォーマットのタスクを記載したものである。

[@] 進行中のタスク
    /// シングルコメント行
    [ ] 子タスク A（ネストのスペースは通常 4 文字）
        [x] 完了した子タスク
            [~] 廃止したタスク
    [w] 保留タスク @JohnDoe が対応中
[ ] + スターの付いたタスク
    [ ] ! 優先度の高いタスク
        [ ] !! さらに優先度の高いタスク
            [ ] !!! 最も優先度の高いタスク
[ ] タスク
    複数行コメント 1 行目
    複数行コメント 2 行目
    [ ] ネストされたタスク A
        ネストされたタスク A の複数行コメント 1 行目
        ネストされたタスク A の複数行コメント 2 行目
        
        ネストをしていれば、改行して何も記載されていない行も OK（複数行コメント 4 行目）
[ ] タグの付いたタスク #home タグの前後は半角スペースを入れる
[ ] 開始日があるタスク 2025-01-01 -> から開始
    [ ] 期限のあるタスク -> 2025-12-31 までに完了させる
    [ ] 開始日と期限があるタスク 2025-02-01 -> 2025-03-31 内に対応する
[ ] key-value 型のタグのあるタスク #key:value
[ ] 特殊なタグ（id）を使用した依存関係のあるタスク（ユニークの ID を持つ） #id:20250101-0000
[ ] 特殊なタグ（dep）を利用した依存関係のあるタスク（依存先 ID を指定） #dep:20250101-0000
[ ] 依存関係のあるタスク 2（依存先 ID を指定）  #dep:20250101-0000
[ ] 人 @JohnDoe や場所 @home を @person タグで表わすことができる
```


## 3. フォーマット仕様

### 3.1 altodo と Markdown (CommonMark) の関係

altodo は Markdown (CommonMark) 内に埋め込み可能な TODO フォーマットである。
Markdown の他の構文（見出し、リスト、コードブロック等）と衝突しない設計を採用するため、Markdown (CommonMark) のインライン要素を含む Markdown (CommonMark) の構文がそのまま使用できる。
バッククオート ``` によるコード（インライン）やコードブロックに囲まれたタグなど altodo の構文のテキストは、Markdown (CommonMark) のコードと同じ扱いになり、altodo の構文の効果にならない。


### 3.2 タスク行とコメント行（一行・複数）

#### 3.2.1 タスク行と一行コメント行について

タスク行は、必ず `[ ]`, `[x]`, `[@]`, `[w]` `[~]` （総称して __タスク状態マーク__ と呼ぶ）のいずれかで始まり、タスクの状態を示す。
一行コメント行は `///` （__一行コメントマーク__ と呼ぶ）で始まり、タスク行やその他の行に対するコメントとして扱われる。
それぞれのマークの直後は、1 つのスペースが必須。
サブタスクの場合は、行頭からスペースのインデントが入る。

__例__

```
[ ] タスク行
/// これは一行コメント行
```


#### 3.2.2 サブタスクについて

タスク行、一行コメント行にはネスト（子）となるタスク行・一行コメント行であるサブタスクを追加できる。
サブタスクにする場合、前行のタスク行・一行コメント行のインデント + 4 スペースで始まる。
サブタスクは無限にネストできる。

```
[ ] タスク（親）
    [ ] サブタスク（子）
/// 一行コメント（親）
    /// 一行コメント（子）
    [ ] サブタスク（子）
        [ ] サブタスク（孫）
    [ ] サブタスク（子）
```


#### 3.2.3 複数行コメントについて

複数行コメントは、タスク行もしくは一行コメント行に付随する複数行に渡るコメント行である。

- 複数行コメントの開始の行は、前行のタスク行・一行コメント行のインデント + 4 スペースで始まる。
- 複数行コメントの続きの行は、前行の複数行コメントと同じインデントである。
- 複数行コメントはインデントなし行で終了する。

__例__

```
[ ] タスク
    タスクに対する複数行コメント 1 行目
    タスクに対する複数行コメント 2 行目
    
    タスクに対する複数行コメント 4 行目
    ※ 2 行目と 4 行目の間の行は、前行と同じインデントであるため、複数行コメントの続き
    [ ] サブタスク
        サブタスクにも複数行コメントを指定できる
```


### 3.3 タスク行の種類（オープン、完了、進行中など）

タスク行のタスク状況マークは、以下の種類。

| 種類 | 種別     | 種別分類           | 詳細                           |
|------|----------|--------------------|--------------------------------|
| [ ]  | オープン | オープン（実行中） | 実行前タスク                   |
| [@]  | 進行中   | オープン（実行中） | 現在、進行・対応中のタスク     |
| [w]  | 保留     | オープン（実行中） | 保留（待ち状態）のタスク       |
| [x]  | 完了     | 終了               | 完了（実行後）タスク           |
| [~]  | 廃止     | 終了               | 完了する前に廃止となったタスク |


### 3.4 フラグについて

タスク行のタスク情報マーク、もしくは一行コメント行の一行コメントマークの直後の文字列は __フラグ__ と呼ばれる。
フラグの後には 1 文字スペースが必要。

__例__

```
[ ] + スターのフラグ
[ ] +NG 例の行。スターのフラグを付与したつもりが、1 文字のスペースが足りていない。通常のテキストとなる
```


#### 3.4.1 スター

スター（重要度が特に高い）タスクとなるよう指定が可能である。
フラグを `+` とした場合、スターとなる。スターは特に重要度の高いタスクとなる。

__例__

```
[ ] + スターフラグの付いた特に重要度の高いタスク
```


#### 3.4.2 優先度

重要度に応じたタスクの優先度付けが可能である。
フラグとして `!` を付与した場合、 `!` の文字数が多いほど重要度が高いタスクとなる。
スター `+` と比べ、どちらが優先度が高いかはユーザの任意である。
`!` が 4 個以上はアプリケーション内部では 3 個に切り捨てられ、同等扱いとする。

__例__

```
[ ] ! 重要度が高いタスク
[ ] !! さらに重要度が高いタスク
[ ] !!! よりさらに重要度が高いタスク
[ ] !!!!!! より、さらに重要度の高いタスクであるが、アプリケーションとしては 3 つの `!` (`!!!`) と同じ扱いをする
```


### 3.5 インライン要素

altodo フォーマット独自のインライン要素を記載する。Markdown のインラインフォーマットは、Markdown の仕様を参照すること。
altodo のインライン要素は、altodo のタスク行、一行コメント行、および Markdown の見出しに埋め込むことができる。
それ以外の箇所に記載しても、通常のテキストとして扱われる。


#### 3.5.1 タグ

SNS のハッシュタグのように、タスクにプロパティを持たせることができる。
タグは `#` で始まり、使用可能文字は ASCII 文字列でも `[-_a-zA-Z0-9]+` の正規表現に一致する必要がある。
ただし、値（バリュー）のみダブルクオート（ `"` ）で囲むことにより、半角スペースを含む ASCII 文字列以外の文字も使用できる。
タグは本文内、または見出し内に埋め込むことが可能。
タスクに複数のタグを指定することは可能。
タグ前後に半角スペース必須。ただし、行末にタグが来た場合は行末のスペースは不要。
`#key:value` の形式で、キーバリュー型のプロパティとすることができる。

__例__

```
[ ] #home でタスクを行う
[w] 仕様書の提出待ち #work-by:john
[x] #work #company #clock:8AM の定例タスク
[ ] 行末にタグを指定した場合は、行末のスペースは不要 #end-of-line
[ ] いわゆる 2 バイト文字を値にしたい場合は #char:"日本語" ダブルクオートで囲む
[ ] ダブルクオートで囲めば #tag:"space も含む" ことができる
```


#### 3.5.2 開始日・期日

タスクの開始日・期日を指定できる。日付表記は `YYYY-MM-DD` 形式とするが、任意のフォーマットも指定可能である。例えば `YYYY/MM/DD` 形式など。
開始日・期日の前後に半角スペース必須。ただし、行末にタグが来た場合は行末のスペースは不要。

- 開始日の指定 `YYYY-MM-DD ->`
- 期日の指定 `-> YYYY-MM-DD`
- 両方の指定 `YYYY-MM-DD -> YYYY-MM-DD`

```
[ ] 2000 年 6 月 1 日から開始するタスク 2000-06-01 ->
[ ] 2001 年 12 月 31 日までの期日のタスク -> 2001-12-31
[ ] 2002 年 1 月 1 日から開始であり、期日が同年 3 月 31 日のタスク 2002-01-01 -> 2002-03-31
    [ ] このタスクは -> 2002-02-15 までに完了させる、など、どこに記載してもいい


## M 案件 2003-04-01 - > 2004-03-31

見出しにも適用が可能
```


### 3.6 特殊なタグについて

タグが特殊なプロパティを持ち、タスクに対して特殊な効果を表すタグについて記載する。


#### 3.6.1 ID #id

任意長のユニーク識別子として ID を付与したタグを作成できる。
タスク行に `#id:[value]` を付与し、他のタスク行が同 ID を参照可能である。
`[value]` は任意の値でよいが、通常ユニークな ID を付与するべきである。

__例__

```
[ ] このタスクは #id:20010101_abc である
[ ] UUID #id:7d444840-9dc0-11d1-b245-5ffdce74fad2 でもよい
```


#### 3.6.2 依存関係 #dep

依存関係を示すタグを作成できる。依存関係のあるタスクにおいて、依存するタスクは依存先が完了しなければ実行ができないという意味になる。
`#dep:[value]` で参照先 ID の value を指定する。
存在しない value の ID を指定することは可能であるが、依存関係は設定されず依存関係のないタスクとなる。

```
[ ] ID が付与されたタスク #id:20030120_abcd
[ ] 依存先のあるタスク（依存するタスク） #dep:20030120_abcd
[ ] 依存先の指定があるが、依存先がないタスク。孤立しているため、依存関係がない #dep:19990101_xyz
```

依存先がない孤立したタスクが発生する理由は、依存先が完了しリストから削除されるなどが考えられる。



#### 3.6.3 担当者・場所タグ @person

人や場所を表す特殊なタグ。`@` の後に任意の文字列（スペース除く）を指定する。
タグ前後に半角スペース必須。ただし、行末にタグが来た場合は行末のスペースは不要。

__例__

```
[ ] タスク @smith に確認
[ ] 会議 @田中 @会社 2026-02-15
[ ] 買い物 @school の近く
[ ] レビュー依頼 @john @mary
```

__用途__

- 担当者の指定
- 場所の指定
- フィルタ条件として使用（将来実装予定）


#### 3.6.4 タスクが完了・廃止された時刻 #done

タスクが完了・廃止された時刻を記載するタグ。タグの付与は任意である。
※ アプリケーション側が自動で付与する想定。

value はデフォルトでは ISO 8601 形式（`YYYY-MM-DDTHH:MM:SS+HH:MM`）。
`YYYY-MM-DD` もしくは unixtime なども許容する。ユーザの任意でよい。
再度タスクがオープンになった場合は `#done` タグを削除すること。

__例__

```
[x] ドキュメント作成 #done:2003-01-15
    [~] レビューを実施する #person:john #done:1042624800
```
